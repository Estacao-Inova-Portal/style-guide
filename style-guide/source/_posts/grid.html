O grid responsivo se adapta a tela do usuário, seu tamanho e orientação, garantindo consistência entre diferentes layouts

Como funciona

O grid é baseado no módulo de layout "Flexible Box", conhecido como Flexbox. O elemento bloco .grid tem um elemento principal, .grid__container e juntos eles formam a base desse sistema.

O primeiro passo é adicionar a classe .grid no elemento de hierarquia mais alta dentro da sua folha de marcação (HTML), esse provavelmente sendo o body.

O segundo passo é adicionar o elemento de conteiner no elemento que funcionará como container no conteúdo, a sugestão é que esse seja o main.

<!DOCTYPE html>
<html lang="pt-br">
<head>
  ...
</head>
<body class="grid">
  <main class="grid__container">
    ...
  </main>
</body>
</html>

Pronto! Seguindo esse esqueleto você acaba de montar uma página responsiva com o grid padrão.

Entendendo o Container

O container tem uma largura de 244 unidades padrão (4.5px x 244 = 1098px). Assim ele pode comportar duas colunas de texto com tamanho mínimo de coluna e a calha das colunas (espaço de respiro entre elas).

Para saber mais sobre unidade mínima padrão e larguras de coluna acesse a seção Tipografia

Por padrão o container se encontra centralizado na página, mas existem classes modificadoras que mudam esse comportamento, mais sobre elas pode ser encontrado abaixo no capítulo sobre modificadores na seção sobre o modificador Grid sem margem.

Os elementos do Grid

Além do elemento de container, existem mais alguns elementos no grid:

  • Elemento de menu .grid__menu

  • Elemento de coluna .grid__columns

O menu lateral

O elemento .grid__menu deve ser filho direto do .grid ou seja, estar no mesmo nível do .grid__container. É com esse elemento que é montado o menu ao lado nesse Guia de estilos por exemplo.

<body class="grid">
  <aside class="grid__menu">
    ...
  </aside>
  <main class="grid__container">
    ...
  </main>
</body>

O estilo do menu e seus elementos é definido pelo componente Menu.

Layout de múltiplas colunas

Para criar um layout de múltiplas colunas o primeiro passo é criar um elemento com a classe .grid__columns, ele será o pai dos elementos que estarão em colunas.

A calha (distância entre colunas) é sempre de 27px, o mesmo tamanho base da altura de linha.

Por padrão o grid força quantos elementos você colocar dentro de uma única linha. Existem modificadores que mudam esse comportamento.

Eu sou um texto que está utilizando uma coluna. Tenho também mais elementos aqui comigo, como um botão por exemplo.

Ok

Eu sou uma coluna mais simples e todo o meu conteúdo se resume a apenas texto...

  <section>

Modificadores do Grid

Grid Wrap

Por padrão o .grid__columns vai forçar todos os seus filhos diretos dentro de uma mesma linha se estiver renderizando em dispositivos de tela grande. Esse modificador força o grid a separar esses elementos em múltiplas linhas, respeitando o tamanho atribuido para esses blocos/elementos.

Importante lembrar que no comportamento padrão do .grid__columns todos os seus filhos diretos vão ter a mesma largura, ao aplicar o .grid--wrap eles poderão ter tamanho variável, caso não tenham uma largura fixa atribuída.

Grid sem Wrap

Tem o efeito contrário do modificador anterior. O caso de uso principal do .grid--nowrap é quando se deseja forçar, mesmo em dispositivos móveis, que algum conteúdo seja apresentado em colunas.

Grid sem Margem

O .grid-no-margin pode ser aplicado no .grid__container. Por padrão o container está centralizado na tela, ao aplicar esse modificador ele ficará justificado à esquerda, como no caso desse Guia de estilos.

Elementos com alturas variáveis

Por padrão, todos os elementos dentro de .grid__columns terão a mesma altura do maior elemento. Ao adicionar o modificador .grid--align-start esse comportamento muda e os elementos tem sua altura inicial respeitada.

Cores de fundo

O modificador .grid--background-light aplica a cor cinza claríssimo como a cor de fundo do grid.

Conteúdo em coluna

Com .grid--column você força o elemento de grid a seguir a direção de coluna, e não o padrão de linha. Útil para páginas que não tem elementos ou menus laterais.

Sem grid--column

Sou um elemento Sou outro elemento Sou mais um elemento

Com grid--column

Sou um elemento Sou outro elemento Sou mais um elemento

Tamanho ideal de texto

Como pode ser visto na página de Tipografia, existe um tamanho ideal de texto, e ele é uma coluna de até 170 unidades básicas de largura, ou seja 765px.

Utilize o modificador .grid--ideal-type para garantir que a coluna de texto respeitara exatamente o tamanho ideal de linha.

Modificadores para dispositivos móveis

Scroll de linha única mobile

Vamos pegar o mesmo exemplo com cards que utilizamos no .grid--wrap, mas agora vamos também adicionar a classe .grid--overflow-mobile.

O que acontece aqui é que, em dispositivos móveis, os cards estão sendo apresentados na mesma linha, com uma barra de rolagem ajudando na navegação e garantindo que eles tenham o seu tamanho mínimo respeitado.

Enquanto isso, em dispositivos de tela grande eles seguem o padrão apontado pelas classes .grid__columns e .grid--wrap.
